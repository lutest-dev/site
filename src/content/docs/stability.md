---
title: Stability
---

Lutest is still early.

You should treat the current surface as usable, but not stable.

That is the main thing to keep in mind while reading the rest of the docs. The model is real. Some edges are still moving.

## What already feels solid

- co-located test authoring
- the Lutest package as the authoring API
- `lutest.toml` as the configuration surface
- `lute` as the runtime you can use today

## What may still change

- CLI shape
- reporter output
- configuration depth
- runtime coverage beyond `lute`

Breaking changes are still acceptable at this stage.
